\section*{Problem 1}

In order to test that the vectors produced by my code were actually
eigenvectors of $\m{A}$, I verified that for
$\epsilon' \approx \epsilon$ given as a parameter
\al{
\sum_i \lp (\m{A}v)_i - \lambda v_i \rp^2 < \epsilon'
}

The following bash commands are an overview of how to reproduce the results that follow. 

Figure~\ref{plot1} demonstrates the pattern of convergence given
varialbes $\epsilon$ and $N$ for matrix $\m A$. As one might expect,
the number of iterations required for the power method to converge is
proportional to the selected precision.  However it is also
interesting to note that the number of iterations to
convergence \emph{decreases} as the matrix size increases. 

\figH {plot1.png}{Plot of convergence iterations vs $\epsilon$ for
       selected matrix sizes}{plot1}{.8}

It can be
noted that the number of iterations grew logarithmically with
$\epsilon$, i.e. for the 10$\times$10 matrix,
\al{
\text{Iterations} \approx -0.78*\log(\epsilon)-0.16
}

with an $r^2 = .9989$. Furthermore, as a function of matrix size for a
fized $\epsilon$, the number of iteration necessary as a function of matrix size $N$ was close to the following,
\al{
 \text{Iterations} \approx 49.5\times N^{-.43} + 9
}

Therefore, we can say that as a function of $\epsilon$, convergence is
on the order of $\mathcal{O}(\log(\epsilon))$.

Furthermore, I implemented the Jacobi algorithm.  Although it was able
to find the correct solution, it did using more iterative cycles.
Given that each cycle required work to find the next pivot, and update
the Jacobi matrix according, it also required more work per
cycle. Therefore, the performance Jacobi implementation was below that
of the power method.

\section*{Part 2}

The margin of error for the reconstructed matrix falls quadratically
as $k$ increases.  More specifically for a $10\times20$ matrix, the
Frobenius norm followed the function
\al{
    \text{Error} = y_1 \approx  2.48(k - 10.69)^2 - 9
}

\figH {plot2.png}{Plot of Frobenius norm vs $k$ precision
       level $\epsilon = 1\times 10^{-10}$ }{plot2}{.8}

This is exactly as one might expect as use find more information we
expect our model to be better. Since the eigenvectors are orthogonal,
each new eigenvector used in reconstructing the matrix adds unique
information.

\section*{Part 3}

In Figure~\ref{plot3}, we can see two trends, (1) a general trend that
the error decreases as the probability that each element $\m A_{ij}$
will be uncorrupted increases, the margin of error between the
reconstructed matrix $\tilde{\m {A}}$ decreases. Furthermore, (2) the
error is smaller for smaller values of $k$.

\vspace{20pt}\figH {plot3.png}{Plot of Frobenius norm vs $k$}{plot3}{.8}

At first it didn't make sense to me that the accuracy would decrease
 as $k$ increased.  This was because I was overlooking the fact that
 $k$ was not only used to reconstruct the matrix, but also to
 determine its rank in the first place, and increasing, in a way, the
 entropy of the system.
